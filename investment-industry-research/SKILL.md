# investment-industry-research — 投资导向行业研究

## 定位

本 Skill 生成可用于**授信审查背景章节**、**债券/股权投资行业研究报告**的系统性行业分析，具有：

- **权威来源优先**：官方统计 > 行业协会报告 > 上市公司年报 > 研究机构报告 > 媒体资讯
- **事实/推断/观点分离**：严格按 `work-analysis-loop` 六层数据分级
- **产业链全景**：上游原材料/设备 → 中游制造/服务 → 下游应用/渠道 → 终端客户
- **技术路线图谱**：主流技术路线、技术代际、关键工艺节点
- **公司竞争映射**：主要玩家、市场份额、差异化竞争优势
- **投资逻辑提炼**：核心驱动因素、关键风险、催化剂事件

## 研究模块结构

### 模块 1：行业定义与边界

| 要素 | 内容 |
|------|------|
| **行业名称** | 中文名称 + 英文名称 + 常用缩写 |
| **监管分类** | 国民经济行业分类（GB/T 4754）/ 证监会行业分类 |
| **核心产品/服务** | 主要品类，含技术路线 |
| **市场规模** | 总量（亿元）、增速（CAGR）、数据年份、来源 |
| **产业链位置** | 上游/中游/下游，各环节价值分布 |

### 模块 2：产业链分析

```
产业链图谱（文字版）：

上游（原材料/设备/零部件）
├── [类别1]：主要供应商、国产化率、价格趋势
├── [类别2]：...
└── 关键瓶颈：卡脖子环节/进口依赖

中游（制造/加工/服务）
├── [细分1]：市场规模、集中度（CR5）、代表企业
├── [细分2]：...
└── 关键竞争要素：技术/成本/资质/客户关系

下游（应用场景/终端）
├── [场景1]：需求规模、增速、对中游议价能力
├── [场景2]：...
└── 需求驱动因素
```

### 模块 3：技术路线图谱

| 维度 | 描述 |
|------|------|
| **主流技术路线** | 当前主流 + 潜在替代路线 |
| **技术代际** | 第几代技术，当前处于哪个阶段 |
| **关键工艺节点** | 决定产品性能的关键工艺 |
| **技术壁垒** | 主要壁垒类型（专利/工艺know-how/设备/人才）|
| **国产化进度** | 各环节国产化率 + 主要替代路线 |
| **下一代技术** | R&D 方向，预计商业化时间 |

### 模块 4：市场格局与竞争分析

```markdown
## 市场集中度
- 全行业 CR5/CR10：XX%（来源：行业协会报告/年份）
- 头部企业市占率：
  | 企业 | 市占率 | 上市状态 | 代码 |
  |------|--------|---------|------|
  | ...  |        |         |      |

## 竞争格局特征
- [集中型 / 分散型 / 双寡头 / 多强并立]：原因分析
- 主要竞争维度：[价格 / 技术 / 产能 / 渠道 / 资质]
- 近三年格局变化趋势：[整合 / 分散 / 新进入者冲击]

## 代表企业简析
（每家 3-5 行，标注数据来源年份）
```

### 模块 5：政策与监管环境

| 要素 | 内容 |
|------|------|
| **主管部门** | 国家级 + 省级主管部门 |
| **核心政策文件** | 政策名称 + 发布日期 + 关键条款 |
| **准入资质** | 生产/经营必要资质，取得难度 |
| **环保/安全要求** | 主要约束，达标现状 |
| **产业政策方向** | 鼓励/限制/替代，对行业影响 |
| **补贴/税收优惠** | 种类、规模、期限 |

### 模块 6：供需分析与景气周期

```markdown
## 供需平衡
- 供给：现有产能（万吨/GW/亿元）+ 在建产能 + 规划产能
- 需求：当前需求量 + 驱动因素分解 + 预测增速
- 供需缺口：过剩/紧张/平衡，影响因素

## 景气周期
- 当前处于周期哪个阶段（复苏/景气/衰退/底部）
- 核心判断依据（库存水平/价格走势/订单情况）
- 历史周期规律（上行多少年/下行多少年）

## 价格与盈利周期
- 关键产品/服务价格走势（最近3年）
- 行业平均毛利率/净利率走势
- 价格变动的主要驱动因素
```

### 模块 7：公司竞争映射与投资建议

```markdown
## 投资主题
[核心驱动因素，2-3个句子，明确标注为 [推断] 或 [观点]]

## 催化剂事件
- 短期（0-6个月）：[具体事件，如新订单/政策落地/技术量产]
- 中期（6-18个月）：[具体事件]
- 长期（1-3年）：[趋势性变化]

## 关键风险
(1) [风险1]：描述 + 发生概率评估 + 影响程度
(2) [风险2]：...
(3) [风险3]：...

## 主要上市公司映射
| 企业名称 | 代码 | 细分定位 | 研究关注点 |
|---------|------|---------|-----------|
| ...     |      |          |           |

注：本部分为 [推断] 和 [观点] 层级，基于公开信息分析，不构成投资建议。
```

## 权威来源优先级

| 优先级 | 来源类型 | 说明 |
|:------:|---------|------|
| P1 | 政府官方统计 | 国家统计局、工信部、能源局、发改委等 |
| P2 | 行业协会报告 | 中国汽车工业协会、中国半导体行业协会等 |
| P3 | 上市公司年报/公告 | 经审计，最高可信度的一手财务数据 |
| P4 | 专业研究机构 | 中国信通院、CCID、CREA、IEA 等 |
| P5 | 头部投行/研究所报告 | 中金、华泰、天风、国泰君安等，需注明利益冲突 |
| P6 | 财经媒体/专业媒体 | 21世纪、财新、证券时报等，用于近期动态 |

**禁止使用**：未经核实的微信文章、个人公众号、小红书等；数据超过3年的研究报告（特别是技术领域）。

## work-analysis-loop 集成

本 Skill 的每个分析模块作为独立的 Loop Round 执行：

| Round | 模块 | 主要任务 | 验证点 |
|-------|------|---------|--------|
| 1 | 行业定义与市场规模 | 确认边界、获取规模数据 | 来源权威性、数据时效性 |
| 2 | 产业链图谱 | 上中下游结构化 | 覆盖完整性、关键节点是否遗漏 |
| 3 | 技术路线 | 主流路线、代际分析 | 技术细节是否有一手来源支撑 |
| 4 | 市场格局 | 竞争结构、企业简析 | CR指标来源、市占率数据年份 |
| 5 | 政策监管 | 关键政策梳理 | 政策文件是否官方来源 |
| 6 | 供需与景气 | 供需平衡、价格趋势 | 数量数据来源、预测假设是否合理 |
| 7 | 公司映射与投资逻辑 | 综合判断 | 推断是否有足够事实支撑 |

## 行业分析模板库（快速参考）

### 制造业/工业行业

关注重点：产能利用率、原材料价格、技术路线代差、海外竞争压力、补贴退坡

### 消费品行业

关注重点：需求弹性、品牌力、渠道结构、利润率保护能力、复购率

### 科技/半导体行业

关注重点：制程节点、国产化率、客户集中度、研发费用率、专利壁垒

### 能源/公用事业行业

关注重点：电价/气价机制、容量利用率、资本支出周期、政府补贴依赖度

### 金融/地产行业

关注重点：信用质量、资产负债表杠杆、监管政策边际变化（⚠️ 不分析政治因素）

## Trigger（触发条件）

以下情况**必须触发**本 Skill：


| 编号 | 触发条件 | 优先级 |
|------|---------|---------|
| T1 | 用户说出"行业研究"/"产业链分析"/"行业报告" | P0 |
| T2 | 用户说出"投资分析"/"赛道分析" | P0 |
| T3 | 用户提供了行业名称 + 分析意图 | P1 |
| T4 | 用户说"技术路线"/"竞争格局"/"投资建议" | P1 |

排除条件：
- E1: 未指定行业 → 先询问
- E2: 说"分析某公司"→ 触发企业分析类Skill

触发后动作：
1. 识别任务类型、企业/项目/行业、交付物要求
2. 输出任务卡（参考 `work-analysis-loop` 的「快速启动模板」）
3. 请用户确认任务分解
4. 用户确认后，创建 `.state.json`，开始第一轮执行

## State 持久化（断点恢复）

每轮执行结束后，必须将状态写入工作区 `.state.json`，支持中断恢复。

### 写入时机

| 时机 | 说明 |
|------|------|
| 每轮 Round 结束后 | 验证完成、状态已更新 → 写入 |
| 人工决策后 | 用户确认后立即写入 |
| 触发停止条件时 | 写入最终状态 → 生成 Handoff |

### 读取恢复逻辑

启动 Skill 时：
```
  if .state.json 存在：
    → 读取 state，检查 updated_at
    → 输出："检测到未完成任务，是否继续？"
    → 用户确认 → 从 state.current_step 或 stages_pending[0] 继续执行
    → 用户拒绝 → 备份 .state.json 为 .state.json.archived，从头开始
  else：
    → 从头开始，生成新 task_id
```

### State Schema

使用统一 State Schema（参见 `loop-engineering` 的 `references/state-schema.json`）。


### State 字段（本 Skill 特定扩展）

| 字段 | 类型 | 说明 |
|-------|------|------|
| `current_step` | int | 七步框架当前步骤编号（1-7） |
| `steps_completed` | list[int] | 已完成步骤编号 |
| `industry_chain` | object | 产业链图谱缓存 |
| `companies_covered` | list[string] | 已覆盖的上市公司列表 |

### 写入原子性

写入序列：
  1. 将 State 写入 `.state.json.tmp`
  2. `os.replace('.state.json.tmp', '.state.json')`
  3. 确保写入成功（fsync）

## Handoff 格式（停止时生成）

每次触发停止条件（S1-S7）时，必须生成两个文件：

### 1. `.handoff.json`（机器可读）

结构参见 `loop-engineering` Skill 的 Handoff 格式说明。

关键字段：
- `handoff_id`：格式 `HO-MMDD-XXX`
- `stop_reason`：停止原因（S1-S7）
- `state_snapshot`：当前状态快照
- `deliverables`：交付物清单（路径 + 状态）
- `evidence_summary`：证据概况
- `open_issues`：未解决问题列表
- `next_steps`：后续步骤建议

### 2. `HANDOFF.md`（人类可读）

包含：
- 交接ID、时间、停止原因
- 当前状态（阶段、迭代轮次、完成度）
- 交付物清单（表格形式）
- 证据概况（来源数、证据条数、完整度）
- 未解决问题（带建议）
- 后续步骤

### 交接后动作

| 停止原因 | 动作 |
|---------|------|
| S1（完成） | 归档 .state.json，保留 .handoff.json |
| S2/S3/S4/S5 | 保留 .state.json（可恢复），生成 .handoff.json |
| S6（破坏性操作） | 不执行，等待用户显式确认 |
| S7（用户主动停止） | 保留 .state.json（可恢复） |
